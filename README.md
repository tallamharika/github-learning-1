## 🚀 About this exercise

This is an interactive GitHub Skills exercise designed to help you learn the basics of GitHub.

You will practice:

- Creating a branch
- Editing files
- Making commits
- Opening a pull request
- Merging changes

---

## 📚 What you are learning

### 1. Repositories
A repository is where your project lives. It contains files, folders, and history.

### 2. Branches
Branches let you work on different versions of a project safely.

Example:
- `main` → original code
- `my-first-branch` → your changes

### 3. Commits
A commit saves your changes with a message.

Example:
- `added profile update`
- `fixed typo`

### 4. Pull Requests
A pull request is how you suggest changes to be added into the main project.

---

## 🎯 Your task flow

1. Create a branch
2. Edit a file
3. Commit changes
4. Open a pull request
5. Merge pull request

---

## 💡 Tips

- Always check which branch you are working on
- Commit often with clear messages
- Don’t be afraid to make mistakes — that’s how you learn
